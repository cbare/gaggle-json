Gaggle-JSON

Gaggle-JSON is a dialect of JSON in which systems biological data is represented by a handful of data types - list, matrix, network, tuple, and table.

This expands on the capabilities of the original Gaggle adding language independence, flexibility, and easy integration with modern ReSTful web architectures. By concentrating on the definition of data structures and remaining agnostic (like SOAP) of the transport protocol (HTTP, AMQP, etc.), we preserve flexibility for the developer. Because Gaggle-JSON is simply a dialect of JSON, it works across languages and new data types can easily be added without recompiling, a major drawback of the Java-RMI approach.
